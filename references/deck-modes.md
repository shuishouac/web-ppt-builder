# Deck Modes

Choose this before theme selection.

This is not just a visual preference.
It changes the information architecture of the deck.

## 1. `content-dense` | 内容详实型

Use this when the deck should be understandable even if the audience only reads the slides.

Typical characteristics:

- higher information density
- more explanatory text
- more complete evidence and context on-page
- weaker dependence on live narration
- fewer decorative visuals
- images used to support credibility or explanation

Good fit:

- company introductions
- partnership decks
- proposals
- training decks
- operational reviews
- product walkthroughs that need to stand alone

User-facing description:

- “这是一种可独立阅读的 deck。即使用户不听讲解，只看页面内容，也能完整理解你的重点和结论。”

## 2. `design-led` | 设计优先型

Use this when the deck is mainly supporting a live presentation, keynote, launch, or narrative talk.

Typical characteristics:

- lower text density
- more summary-level copy
- stronger use of hero visuals
- stronger dependence on narration
- more emphasis on pacing, contrast, and transitions
- images and motion carry more of the experience

Good fit:

- keynote-style talks
- launch decks
- founder presentations
- event presentations
- public-facing showcase decks

User-facing description:

- “这是一种更像发布会或舞台演示的 deck。页面上只保留关键句和重点词，大部分信息通过图片、节奏和讲解来传达。”

## 3. Decision Rule

If the user is unsure, ask:

1. “你希望观众只看 deck，就能完整理解内容吗？”
2. “还是你会亲自讲，这套 deck 主要是帮你做舞台呈现和节奏推进？”

Decision:

- if the answer is “必须能独立阅读” -> choose `content-dense`
- if the answer is “主要配合我讲” -> choose `design-led`

## 4. Hybrid Rule

If the user wants both:

- default to `content-dense` for business decks
- default to `design-led` for public talks

Then borrow selectively:

- give `content-dense` decks stronger section covers and better imagery
- give `design-led` decks 1 to 2 appendix or backup slides for detailed proof
