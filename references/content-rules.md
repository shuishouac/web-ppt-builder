# Content Rules

Treat this as a web deck, not a document chopped into pages.

## Deck Modes

Read [deck-modes.md](deck-modes.md) before deciding slide density.

### `content-dense`

- Use when the deck should remain understandable even without a presenter.
- Prefer more explanatory text, denser evidence, and stronger self-contained logic.
- Use more pages only when the content cannot be simplified without losing meaning.
- Good for:
  - company profiles
  - proposals
  - product walkthroughs
  - investor or partner briefs

### `design-led`

- Use when the deck supports a live talk, demo, or keynote-like presentation.
- Prefer short headlines, sparse copy, strong visuals, and clearer emotional pacing.
- Let narration carry the detailed reasoning.
- Good for:
  - launch talks
  - founder presentations
  - conference decks
  - public-facing showcase decks

## Slide Density

- Keep most slides to one core idea.
- Keep headlines short enough to scan in under 2 seconds.
- For `content-dense`, prefer 4 to 7 bullets or short explanatory blocks when needed.
- For `design-led`, prefer 1 to 3 bullets and let the image or headline do most of the work.
- Split dense reasoning into multiple slides instead of shrinking text.

## Structure

Default flow:

1. cover
2. problem / tension
3. key insight
4. supporting proof or framework
5. example / demo / comparison
6. recommendation or CTA

In `content-dense`, add more support slides between steps 3 to 5.
In `design-led`, compress those middle sections and preserve momentum.

## Visual Rhythm

Alternate between:

- full-bleed statement slides
- split text / visual slides
- card grids
- section breaks
- stat or quote slides

Do not repeat the same layout five times in a row.

In `content-dense`, the rhythm can be steadier and more document-like.
In `design-led`, the rhythm should feel more cinematic and contrast-heavy.

## Browser-First Rules

- Assume the deck will be shared as a link.
- Avoid tiny text that only works on a projector.
- Use clean spacing and large contrast blocks.
- Prefer screenshots, diagrams, or simple icons over decoration.
- In `content-dense`, optimize for slower reading and clarity.
- In `design-led`, optimize for fast scanning and stage impact.

## Image Rules

- Every image should earn its place.
- If an image is there only to make the slide feel less empty, consider a typographic or diagram-first layout instead.
- Use real photos for real claims whenever credibility matters.
- Use AI images only when the slide is explanatory, atmospheric, or conceptual.
- Do not present AI-generated executive or team portraits as real company photography.
- If a user provides real product or people images, preserve identity and recognizability over visual stylization.
- When using screenshots, crop tightly and remove irrelevant browser or desktop clutter unless the interface chrome is part of the point.
- If an image cannot survive being shown large, do not use it as a hero visual.

## Motion Rules

- Use click reveals only when the order matters.
- Use section transitions to mark narrative changes.
- Do not animate every element.
- If motion does not clarify meaning, remove it.
- In `content-dense`, motion should stay restrained.
- In `design-led`, motion can be more visible, but still needs to support the story.

## Brand Rules

- If a logo is present, keep it subtle and consistent.
- If a headshot is present, use it only on the cover or closing slide unless the deck is personality-led.
- If the user provides brand colors, honor them over preset defaults unless contrast fails.
